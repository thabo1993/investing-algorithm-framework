## Dependencies & parallelization
- **Setup → Foundational → User Stories → Polish.** Phases are ordered; the checkpoints are gates.
- Within a phase, **[P]** tasks touch different files and may run together.
- Across stories: once Foundational is done, US1/US2/US3 are independent and can be parallelized
  across people/agents — implement in priority order if sequential.
- A user story's acceptance test (the `[P]` test task) should exist before its implementation tasks.
