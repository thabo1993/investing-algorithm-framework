## Phase 4 — Backtesting Enhancements

*All tasks in this phase are parallel — each creates a new file in `domain/backtesting/` with no interdependencies. All depend on Phase 1 being complete (use domain models).*

- [ ] **P4.1** [P] Implement backtest market calendar filter
      (`investing_algorithm_framework/domain/backtesting/forex_market.py`)
      **Depends on:** Phase 1 complete
      **Accept:**
      - New module with a function `apply_market_calendar_filter(df: pl.DataFrame, market: str) -> pl.DataFrame`
      - When `market == "FOREX"`, strips weekend candles using `ForexMarketCalendar.filter_market_hours()`
      - Safe for non-forex markets: returns data unchanged
      - Logs warning if >10% of rows filtered (data quality signal)
      - Integrates into backtest data preparation pipeline

- [ ] **P4.2** [P] Implement pip-based P&L calculation
      (`investing_algorithm_framework/domain/backtesting/pip_pnl.py`)
      **Depends on:** Phase 1 complete
      **Accept:**
      - New module with `calculate_pip_pnl(entry, exit, pair, units, lot_type, quote_to_base_rate)` function
      - Formula: `pip_movement = abs(exit - entry) / pip_size`, `pip_pnl = pip_movement * pip_value * lot_size_multiplier`, `base_currency_pnl = pip_pnl * quote_to_base_rate`
      - Returns `PipPnLResult` dataclass with `pips` and `base_currency_pnl` fields
      - All arithmetic uses `Decimal`
      - Unit-tested for EUR/USD, USD/JPY, and GBP/JPY (cross pair)

- [ ] **P4.3** [P] Implement swap cost deduction tracker
      (`investing_algorithm_framework/domain/backtesting/swap_tracker.py`)
      **Depends on:** Phase 1 complete
      **Accept:**
      - New `SwapTracker` class
      - At each day's rollover time (configurable, default 22:00 UTC), checks open positions
      - If position held through rollover, applies `SwapRate.calculate_swap()`
      - Wednesday → triple swap
      - Deducts/credits swap from position P&L
      - No swap cost when `SwapRate` not configured (zero by default)
      - Rollover hour configurable (DST-safe via UTC + US/Eastern offset)

- [ ] **P4.4** [P] Implement margin utilization reporting in backtest metrics
      (`investing_algorithm_framework/domain/backtesting/margin_utilization.py`)
      **Depends on:** Phase 1 complete
      **Accept:**
      - New module with `calculate_margin_utilization(position_notional, account_balance, leverage)` function
      - Returns percentage (float) and absolute margin used (Decimal)
      - Integrates into backtest metrics generation — `BacktestMetrics` includes `max_margin_utilization`, `avg_margin_utilization`
      - Existing non-forex backtests set these to None (backward compatible)
      - All arithmetic uses Decimal
      - Unit-tested for 1:10, 1:30, 1:100 leverage

**Checkpoint:** Backtesting engine enhanced for forex — weekend candles removed, P&L computed in pips + base currency, swap costs deducted at rollover, margin utilization tracked. Forex backtest accuracy matches broker accounting.
